# Solana Partner Diligence

Use this skill when the user needs a structured read on whether another team, protocol, or service is worth depending on.

## What This Skill Is For

- protocol integration review
- infra provider diligence
- API and data dependency review
- partnership screening
- co-marketing or ecosystem launch prep
- pre-vendor and pre-tooling decisions

## Default Operating Procedure

1. Read the target's docs, repo, status pages, onboarding flow, and public comms.
2. Inspect the diligence flow in [diligence.md](diligence.md).
3. Check the failure patterns in [red-flags.md](red-flags.md).
4. Deliver one of the templates in [outputs.md](outputs.md).

## Decision Standard

Answer these clearly:

- can we trust this enough to depend on it
- what breaks if it fails
- what evidence is missing
- should we integrate now, later, or not at all

## Read These Files When Needed

- [diligence.md](diligence.md): diligence workflow
- [red-flags.md](red-flags.md): common integration and partner failure modes
- [outputs.md](outputs.md): memo and verdict templates
